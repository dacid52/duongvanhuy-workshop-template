**Week 1:** [Review the AWS platform, become familiar with the account environment, define a study roadmap, learn cost-control basics, and complete introductory labs.](1.1-week1/)

**Week 2:** [Study Amazon VPC architecture and related networking components, configure core network resources, secure access with Security Groups and NACLs, launch EC2, and review connectivity options such as VPN, VPC Peering, and Transit Gateway.](1.2-week2/)

**Week 3:** [Deploy EC2 instances across subnets, validate internal connectivity, configure NAT Gateway and EC2 Instance Connect Endpoint, set up a hybrid DNS model with Route 53, and work with AWS Backup for EC2 and EBS.](1.3-week3/)

**Week 4:** [Explore AWS Storage Gateway with Amazon S3 for data synchronization, review cache and access control behavior, and deploy a static website on Amazon S3 with the required public access settings.](1.4-week4/)

**Week 5:** [Deepen practical knowledge of Amazon EC2, practice with Windows Server 2025 and Amazon Linux 2023, review instance selection criteria, strengthen VPC and Security Group settings, and study EBS Snapshot, Systems Manager, and IAM fundamentals.](1.5-week5/)

**Week 6:** [Study IAM users, roles, and policies; manage data on Amazon S3; explore AWS Cloud9, AWS STS, EC2 Instance Profiles, CloudWatch monitoring, and cost tracking with AWS Credits.](1.6-week6/)

**Week 7:** [Learn Amazon RDS, connect a managed database to EC2, monitor resources with CloudWatch Dashboards and Alarms, and review Auto Scaling, Elastic Load Balancing, and performance evaluation on AWS.](1.7-week7/)

**Week 8:** [Continue hands-on work with Amazon S3, refine access policies, deploy a static website, examine Amazon CloudFront for faster delivery, and review backup, synchronization, and recovery capabilities.](1.8-week8/)

**Week 9:** [Study highly available deployment patterns on AWS, continue monitoring with CloudWatch, review CloudTrail event logging, work with Systems Manager, and consolidate the knowledge gained so far.](1.9-week9/)

**Week 10:** [Design the AWS deployment architecture, complete the main topology for compute, database, storage, and networking services, prepare the target environment, and finalize the design documentation.](1.10-week10/)

**Week 11:** [Deploy the application environment on AWS, configure servers, database, and storage services, perform integration testing, resolve deployment issues, and review configuration and security settings.](1.11-week11/)

**Week 12:** [Finalize the deployment, evaluate the implementation results, review performance and configuration status, complete the technical report, and prepare the final presentation.](1.12-week12/)
